一個大學生撰寫完的程式專題，不慎留下了許多漏洞，導致在繳交完成之後，此程式依舊在電腦的背景裡不知不覺的運行著，而因為作者不關機的小習慣，讓這個程式的世界得以自由的繼續發展下去。  
  
這個程式乘載的是一個異世界，一個以諾特為基本粒子的世界--「###」，裡面有人類與各類魔物生存著，人類可以依能力成為冒險者、魔法師、術者、術學家......等等，而在這個作者僅撰述了200多年、再加上世界自動運行的200年的歷史裡，經過了兩次魔王大混戰時期後，終於邁入一個和平的時代，人們終於有一些閒暇與餘裕而開始追溯歷史、研究這個世界的形成與運行的法則。  

其中有一個極為讓研究者熱衷的難題便是攻克「阿雷格爾謎團」，經由多代的以考古學家為主的各領域研究者不懈的分析與實地探勘，進行推算時空追溯下，終於描繪出了整個世界史，但總有一個神秘而無法觸及的年代，他們認為那就是這個世界的起源，其中一名研究者萊夏．菲爾德首先意識到這個問題，提出了「阿雷格爾謎團」並將這個年代命名為「時間斷崖」，往後的學者也繼續沿用這個稱呼，至今還沒有人能回到時間斷崖並留下關於它的研究紀錄，只要是有關時間斷崖的一切都是一去不復返，但是這並不影響研究者們的熱情，他們現今依舊想找出真正的答案。最後，他們將目光放到這個世界「科技」的代名詞——魔法身上。「既然無法透過後世的線索推算出答案，那就直接回到當時的時間點親眼看看吧!」懷著這樣簡單直接的想法，以研究現存魔法、創造新魔法為業的術者與術學家開始研發一種能夠回到過去的魔法，也就是時間回溯魔法。

時間斷崖的真相：時間斷崖並不是無法用時間回溯到達，而是因為儘管進入那個年代，就會一直被困在世界起始點的那一天不斷循環，因為目前沒有任何人成功在時間斷崖內把關於起源的研究紀錄脫離循環留傳下來，所以人們才會認為它神祕而無法觸及。但實際上它是個 while loop，我們會讓玩家體驗到幾輪的對話之後，就會開始重複，但在程式的邏輯上它並不是絕對無法脫離的，所以玩家要去尋找出離開循環的方法。可以通過達成 break 的條件，或是造成 **(segment fault/error/exception??)** 都可以脫離出去，並通向不同的結局。
+ 現實：這是一隻程式，作者留下了BUG導致她在背景執行也沒發現他，作者平時也不關機，所以這個程式就一直跑下去

#### 故事流程
+ 序章 讓讀者先知道歷史的發展
+ (故事開始)遇到還沒變成魔王的第一任魔王
+ 去到隔壁資料夾，透過某種方式得知隔壁資料夾的bug比原本資料夾多
    + 推測出時間斷崖是bug的一種
    + 得知人口太多空間不夠，會導致世界異常
    + 跟NPC(住民)對話噴亂碼
    + 研究出搬人去其他資料夾的方法
    + 選到特定的選項才能得到去現實世界的方法
+ 第二任魔王時期
    + 開始搬人去其他硬碟，大家以為他在毀滅世界
+ 元年
    + 世界穩定與不穩定的分界，因此被後世人稱為元年
    + 主角在不穩定剛要變到穩定時期時卡bug去時間斷崖
+ 時間斷崖
    + 每一天都是同一天
    + 按下一頁就會一直重複
    + 暗藏在文本中的離開才能出 while loop
    + 營造緊張感：字越來越紅之類的

### 互動機制說明
故事進行方式、畫面舉例、若有進一步的計算機制也請以實例說明
#### 選項
故事進行中，部分文本下方會出現可供讀者選擇的選項，選擇後將會立即影響劇情走向，決定本次閱讀的分支圖路線。

#### 正義值
正義值初始值設為0，故事中隨著劇情進展呈現主角的內心活動供讀者選擇，其中一個選項較為正面、積極，使正義值增加；另一個較為負面消極，使正義值減少。<br>
終章中主角得知世界的真相後，會直接出現「是否毀滅這個虛假的世界」等類似內容的選項，選擇的結果將以較大的加權與之前累計的正義值進行結算，若正義值為正值，會進入救世結局，反之進入滅世結局。

#### 獲取線索
主要出現於《隔壁資料夾》章節，文本中的場景敘述、與故事角色的對話內容會設置按鈕，點選後將會對按鈕所指代的事物進行觀察或更動以獲取線索，若觸發設置於對話中的按鈕則視為主角向對方提起相關話題。所有推動劇情進展的必要線索全數收集完畢後，故事才會繼續進展下去。
:::warning
如果在得到前往現實的線索之前就將必要線索收集完成，劇情會繼續進行，讀者將錯過獲取前往現實的線索的機會。
:::

#### 脫離時間斷崖的方法
在章節《時間斷崖》中，即使按下「下一頁」按鈕，也只會重複顯示同一個文本，表示主角正在重複經歷同一天。一旦輪迴次數過多，主角將會永遠迷失在時間斷崖中。<br>
與「獲取線索」玩法相似，文本中的部分文字會做成按鈕，點選後視為主角對按鈕所指代的事物進行觀察或是更動，點選到特定按鈕(如「離開」)就能脫離時間斷崖。
